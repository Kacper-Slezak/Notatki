# 1. History:

- [[Docker]] jeden z pierwszych lepszych umożliwiających conteneryzacje
- Powstał Kubernetes
- Aby wspierac inne conteiner runtime niż [[Docker]] powstało CRI - Container runtime interface
- Docker pracowało poza CRI i miał wsparcie osobne
- I [[containerD]] mógł pracować z CRI jako część [[Docker]]'a
- W końcu [[Docker]] przestał być wspierany
- [[containerD]] może teraz sam być instalowany
